TODO 做一个企业级表单引擎
核心思路：
1.JSON Schema表达能力弱，不适合复杂后台；
2.不是零代码，而是隐藏技术型代码(模板)，让开发人员专注业务实现；
3.蓝图(可视化编辑)可以比作在模板里挖洞，洞填上了，业务逻辑就做好了；
4.蓝图的开发难度极高，专注在模板建设上。让开发人员知晓表单开发的详尽步骤和每一步的操作方法，提供工具和代码段把这个开发过程变的更丝滑。
5.如果把表单创建的过程一步一步用git记录起来，步骤要极其细致，就可以让ai训练成一个表单开发助手。

具体做法：
1.细化表单开发过程：
    1.确定这个表单数据的结构；

    UI，开发难度最低，而易变性最高，主要由开发人员开发；
    
    空数据时的初始化
    半数据时的初始化
    满数据时的初始化

    初始化后的校验
    初始化后的联动

    联动 
    校验
    提交

联动	字段之间相互影响（显隐、值计算、禁用等）	用响应式数据流蓝图表达
校验	同步/异步、跨字段、后端校验	用校验管道蓝图表达
提交	前置校验、数据转换、调用API、后处理	用任务流蓝图表达
